# 何を作るか？(what)

## 方針

- 個人利用のため難しいことはやらない
- 求人要件は複数登録可能。求人ごとに応募書類を複数登録できる

## 主要機能

### 応募書類取り込み・解析

- **応募書類入力**: 生テキストをそのまま貼り付けて登録。フォーマットの事前整形は不要で、AIが自動的に構造化する
- **自動構造化**: 氏名・年齢・最寄り駅・希望単価・経験年数・スキル・資格・職歴・学歴・自己PR を自動抽出しデータ化

### 求人要件取り込み・解析

- **求人要件入力**: 生テキストをそのまま貼り付けて登録。フォーマットの事前整形は不要で、AIが自動的に構造化する
- **自動構造化**: 必須スキル・歓迎スキル・求める人物像・業務内容・単価レンジ・経験年数・ポジションレベル・業界経験・資格要件 を自動抽出しデータ化
- **求人要件のカスタマイズ**: 必須スキル・歓迎スキル・経験年数・給与レンジ・ポジションレベルなどを調整

### AIマッチングエンジン

- **マッチングスコア算出**: 各候補者を0〜100点でスコアリング
- **多軸評価**: スキル / 経験年数 / 業界経験 / ポジションレベル を個別評価
- **必須要件充足チェック**: 求人の必須要件を1件ずつ yes/no で判定し、充足率（例: 3/5件）を算出。未充足の必須要件を一覧表示する
- **AIサマリー生成**: 候補者ごとに「強み・懸念点・面接での確認事項」を自動生成
- **スコア根拠の表示**: 各軸のスコアと、得点を押し上げた要因・不足した要件を明示

### スクリーニング・ランキング機能

- **ランキング**: スコア順に候補者一覧を表示

## 利用フロー

1. **求人要件を登録する** — 求人要件取り込み画面でテキストを貼り付けて登録
2. **応募書類を登録する** — 応募書類取り込み画面で対象求人を選んで登録 → スコア算出が自動実行
3. **結果を確認する** — ランキング画面でスコア順に候補者を確認

> 求人要件が1件も登録されていない状態では、応募書類取り込み画面は「まず求人要件を登録してください」と案内し、取り込み操作をブロックする

## 主要画面

### グローバルヘッダー（全画面共通）

全画面の上部に常時表示される。現在操作対象の求人をアプリ全体で共有するための中枢UI。

**要素**
- アプリロゴ／タイトル（左端）
- 求人セレクター — 登録済み求人をドロップダウンで選択。選択中の求人がアプリ全体のコンテキストになる
- 「＋ 求人を追加」ボタン — クリックで求人要件取り込み画面へ遷移（または取り込みモーダルを開く）
- ナビゲーションリンク — 「応募書類取り込み」「ランキング」（右端）

**ふるまい**
- 求人が1件も登録されていない場合、セレクターは「求人を登録してください」というプレースホルダーを表示し、「＋ 求人を追加」ボタンをハイライトする
- 求人を切り替えると、ランキング画面の候補者一覧が即時に切り替わる
- スコア算出中の求人にはセレクター内でローディングインジケーターを表示する
- セレクターに「現在の求人を削除」メニュー項目を設ける — 選択中の求人を削除できる（確認ダイアログあり）

### 求人要件取り込み画面

**目的**: 求人票テキストを貼り付けてAIに構造化させ、求人要件として登録する

**レイアウト**: 左右2カラム構成
- 左カラム: テキスト入力エリア
    - 大きめのテキストエリア（生テキスト貼り付け用）
    - 「AI解析」ボタン
- 右カラム: 構造化結果表示・編集エリア
    - フィールド一覧: 求人タイトル・業務内容・必須スキル（タグ形式）・歓迎スキル（タグ形式）・求める人物像・単価レンジ（下限・上限）・経験年数（最低年数）・ポジションレベル・業界経験・資格要件
    - 各フィールドは直接編集可能

**アクション**
- 「確定・登録」ボタン — 求人要件を保存
- 「クリア」ボタン — 入力内容をリセット

---

### 応募書類取り込み画面

**目的**: 応募書類（職務経歴書・レジュメ）を貼り付けてAIに構造化させ、候補者データとして登録する

**前提条件**: グローバルヘッダーで求人が選択されていること。求人が1件も登録されていない場合はヘッダーが「＋ 求人を追加」をハイライトするため、このページのフォームはブロックして案内のみ表示する

**レイアウト**: 左右2カラム構成
- 左カラム: テキスト入力エリア
    - 大きめのテキストエリア（生テキスト貼り付け用）
    - 「AI解析」ボタン — クリックでAI構造化を実行、ローディング表示あり
- 右カラム: 構造化結果表示・編集エリア
    - 抽出された各フィールドをラベル付きの入力欄で表示
    - フィールド一覧: 氏名・年齢・最寄り駅・希望単価・経験年数・スキル（タグ形式）・資格・職歴・学歴・自己PR
    - 各フィールドは直接編集可能
    - AI解析前はグレーアウト表示

**アクション**
- 「確定・登録」ボタン — 内容を保存し、バックグラウンドでマッチングスコア算出を起動。登録完了後は「ランキングを見る」ボタンを表示してランキング画面へ誘導する
- 「クリア」ボタン — 入力内容をリセット

---

### ランキング画面

**目的**: 選択した求人に対して、候補者をスコア順に一覧表示し、詳細を確認する

**候補者一覧テーブル**
- カラム: 順位 / 氏名 / 総合スコア / 必須充足率 / スキルスコア / 経験年数スコア / 業界経験スコア / ポジションレベルスコア / 登録日
- 必須充足率は「3/5」のように件数で表示。未充足がある場合は赤色で強調する
- スコア降順でソート（変更不可）
- 各行に「詳細」ボタン・「削除」ボタン

**詳細ダイアログ**
- 基本プロフィールタブ
    - 氏名・年齢・最寄り駅・希望単価・経験年数・スキル・資格・学歴
- スコア根拠タブ
    - 必須要件充足チェックリスト — 必須要件を1件ずつ列挙し、充足 / 未充足を yes/no で表示
    - 各軸のスコアをバーグラフで表示
    - 得点を押し上げた要因（マッチした要件）のリスト
    - 不足した要件のリスト
- AIサマリータブ
    - 強み・懸念点・面接での確認事項を文章で表示
- 応募書類タブ
    - 応募書類の原文を表示し、スコアの根拠を元データで確認できるようにする
