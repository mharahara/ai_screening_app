# API・非同期処理 (how)

## API エンドポイント

| メソッド | パス | 役割 |
|----------|------|------|
| POST | `/jobs/parse` | 求人票テキストを構造化して返す（未保存） |
| POST | `/jobs` | 構造化済み求人要件を保存 |
| GET | `/jobs` | 求人一覧（セレクター用） |
| GET | `/jobs/{id}` | 求人を 1 件取得（全フィールド） |
| DELETE | `/jobs/{id}` | 求人と関連データを削除 |
| POST | `/candidates/parse` | 応募書類テキストを構造化して返す（未保存） |
| POST | `/candidates` | 候補者を保存し、スコア算出をバックグラウンド起動 |
| DELETE | `/candidates/{id}` | 候補者と関連スコアを削除 |
| GET | `/jobs/{id}/rankings` | 指定求人の候補者をスコア順に取得 |
| GET | `/candidates/{id}/detail` | 候補者の詳細（プロフィール・スコア根拠・サマリー・原文） |

## 非同期スコア算出

- 候補者登録時、レスポンスはすぐ返し、スコア算出は FastAPI の `BackgroundTasks` で実行する
- フロントは登録後にポーリングまたは再取得でスコア完了を検知し、ランキングへ誘導する
- 算出中の求人はセレクターでローディング表示する（02_what 参照）
