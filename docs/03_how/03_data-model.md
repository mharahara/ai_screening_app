# データモデル (how)

| テーブル | 主なカラム |
|----------|------------|
| `jobs` | id, タイトル, 業務内容, 必須スキル, 歓迎スキル, 求める人物像, 単価レンジ, 経験年数, ポジションレベル, 業界経験, 資格要件, 原文, 作成日時 |
| `candidates` | id, job_id, 氏名, 年齢, 最寄り駅, 希望単価, 経験年数, スキル, 資格, 職歴, 学歴, 自己PR, 原文, 作成日時 |
| `scores` | id, candidate_id, 総合スコア, スキルスコア, 経験年数スコア, 業界経験スコア, ポジションレベルスコア, required_met（充足件数）, required_total（全件数）, 必須要件チェック(JSON), 強み, 懸念点, 確認事項, 算出日時 |

- スキル・タグ・チェックリストなど配列・構造データは JSON カラムとして保持する
- `candidates` は `job_id` で求人に紐づき、求人削除時は関連する候補者・スコアもカスケード削除する
